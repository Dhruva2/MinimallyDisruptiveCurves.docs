Background: model-fitting

Graphically



(bracketed blocks are optional. Ideal outputs could be experimental data)

But we can abstract away fixed blocks to get this:


Verbally:

  • When we make a model of a system, we want it recreate some desired behaviour(s).

  • To do so, we make a cost (also called loss) function, that takes in model behaviour, and spits out how bad it is. Larger loss <=> less desirable model behaviour.

  • Often, the overall cost function is a sum of sub cost functions. EG we want the model to do nnn different things for nnn different inputs. Each ideal input-output behaviour specifies a cost function.

Mathematically:

  • Vector of model parameters: θ\thetaθ.

  • Space of allowable parameters: Θ\ThetaΘ (∋θ\ni \theta∋θ).

  • Cost function: C(θ)C(\theta)C(θ). Remember that lower cost <=> better fitting parameter vector.


Model fitting (also known as training, optimisation, calibration, ...) is then running:

θ∗\theta^*θ∗ = Minimise C(θ)C(\theta)C(θ)

subject to θ∈Θ\theta \in \Thetaθ∈Θ.


  • Since C(θ)C(\theta)C(θ) is a scalar function (returns a scalar), we can think of it as a landscape, where height is analogous to C(θ)C(\theta)C(θ), and lateral co-ordinates are the parameters of the problem. Then model fitting is just finding a path to the deepest point of the landscape! (see below)

What are we trying to do

Recall the goal. Given a fitted model, we want to:

Find functional relationships between model parameters that best preserve model behaviour.

How do we start? First let's gather up what we obtained from the model fitting step:

  1. A cost function C(θ)C(\theta)C(θ) that maps plausible vectors of model parameter values (θ\thetaθ) to cost (how badly the model performs).

  2. A locally optimal set of parameters θ∗\theta^*θ∗. Basic vector calculus gives: ∇C(θ∗)=0\nabla C(\theta^*) = 0∇C(θ∗)=0. Translation: the slope of a landscape at the deepest point is zero
